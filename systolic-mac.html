overview

Systolic arrays are the core compute primitive behind modern neural network accelerators — Google's TPU, Apple's Neural Engine, and virtually every edge AI chip use variants of this dataflow architecture. This project implements one from scratch in synthesizable SystemVerilog, designed to tape out on the Tiny Tapeout community shuttle.

The design uses a weight-stationary dataflow: weights are loaded into the 4×4 grid of processing elements via SPI, then input activations are streamed row-by-row through the array. Each PE accumulates a partial sum, and the final 4×4 result matrix is read back through SPI. A control FSM handles the three-phase protocol: LOAD → COMPUTE → READBACK.

architecture

Each of the 16 PEs performs a MAC operation each clock cycle: acc += a_in × b_in. Inputs pass through in one dimension; the orthogonal dimension moves to the adjacent PE on the next cycle, creating the characteristic "wave" of computation that gives systolic arrays their name.

code

Each PE is a registered MAC with pass-through — weights are latched once during LOAD, inputs flow through every cycle during COMPUTE.

// Processing Element — weight-stationary MAC
module pe #(
  parameter int DATA_W = 8,
  parameter int ACC_W  = 24
) (
  input  logic                  clk, rst_n,
  input  logic                  load_en,
  input  logic [DATA_W-1:0]    a_in, b_in,
  input  logic [DATA_W-1:0]    weight_in,
  output logic [DATA_W-1:0]    a_out, b_out,
  output logic [ACC_W-1:0]     acc_out
);

  logic [DATA_W-1:0] weight_reg;

  always_ff @(posedge clk or negedge rst_n) begin
    if (!rst_n) begin
      weight_reg <= '0;
      acc_out    <= '0;
      a_out      <= '0;
      b_out      <= '0;
    end else begin
      if (load_en) weight_reg <= weight_in;
      a_out   <= a_in;                          // pass through
      b_out   <= b_in;
      acc_out <= acc_out + (a_in * weight_reg); // MAC
    end
  end
endmodule
