FPGA Vending Machine Controller

SystemVerilog FSM Boolean Board RTL Mealy / Moore Two-flop Synchronizer 100MHz

A SystemVerilog vending machine controller synthesized on a Boolean FPGA board. Accepts nickels (1¢), dimes (3¢), and quarters (5¢) via a physical coin acceptor with pulse-counting detection, input synchronizers for metastability handling, and a Moore FSM that tracks credit and drops the product when sufficient funds are inserted. Credit is displayed live on the 7-segment display; the LEDs light up on a successful vend.

overview

The system accepts three coin denominations via a physical coin acceptor connected to PMOD_A[6], or through manual button presses in debug mode. A pulse-counting FSM decodes the coin type from the number of 25ms pulses on the vend line. The main coin FSM tracks cumulative credit and asserts drop whenever credit reaches 4¢, lighting all 8 LEDs and displaying the remaining credit on the 7-segment display.

fsm design

The myCoinFSM has 8 states (S0–S7) organized into two bands. States S0–S3 represent credit 0–3¢ with drop=0; states S4–S7 represent credit 0–3¢ with drop=1 (vend triggered). Credit is encoded as state mod 4.

Edge detection registers (ce, te, pe) prevent a held coin signal from being counted twice — only the rising edge of each coin pulse advances the state.

coin detection

The physical coin acceptor outputs a normally-high, active-low pulse train on vend_L. Each pulse is ~25ms low with a ~100ms inter-pulse pause. The AutoAccepter decodes coin type by counting falling edges:

  • A 128-bit shift register continuously samples vend_L. When all 128 bits are 0 or 1, the signal is declared stable (fallen or risen) — filtering glitches up to 1.28µs.
  • The AutoAccept_FSM counts fall/rise pairs across 11 states. A 25-bit timeout counter detects the end of a coin's pulse train.
  • 1 pulse → circle (nickel), 3 pulses → triangle (dime), 5 pulses → pentagon (quarter).

In manual mode (SW[15]), BTN presses are routed through the ManualAccepter, which uses a 7-state FSM to generate a single clean pulse per button press and ignore held inputs.

metastability handling

All six async inputs (vend_L, manualMode, manualClock, manualCircle, manualTriangle, manualPentagon) pass through a two-flop synchronizer before entering the 100MHz clock domain. Each synchronizer chains two D flip-flops — the first may go metastable, but the second samples after a full clock period of resolution time, making metastable propagation vanishingly unlikely.

display & outputs

The EightSevenSegmentDisplays module multiplexes 8 digits across two 4-digit display groups using a 17-bit cycle counter (counting to 100,000 at 100MHz = 1ms per digit) and a 2-bit anode index. A one-cold decoder drives the active anode. Only HEX0 (credit value) is active — HEX1–7 are blanked.

The LED output is purely combinational: LD[7:0] = {8{drop}} — all 8 lower LEDs light simultaneously when a vend event is triggered.
